프로젝트 소개

엑셀 수기 관리와 파편화된 이메일 소통의 비효율을, 무역 코어 ERP와 OpenAI 기반 업무 자동화로 해소합니다. 비개발자 담당자도 이해하기 쉽도록 기획 단계부터 친절하게 리딩합니다.

핵심 요구사항 분석

  • 기준정보·대시보드 — 선번(Project)·고객사·제조사 마스터, 사용자 권한(영업/관리자), 월별 선적 스케줄 공유 캘린더, 매입/매출 및 선지급·미지급 구분 대시보드
  • 영업·수출입 물류 — 견적서 작성·이력·PDF, 발주(PO) 등록·진행 자동 추적, 납기 임박/지연 사내 알림, 선적 스케줄 트래킹, Commercial Invoice·Packing List 데이터 연동 자동 생성, 클레임 접수·이력 관리
  • AI·이메일 자동화(OpenAI) — 사내 메일 계정 백그라운드 동기화, 다국어(일/영/중) 메일 분석 및 클레임 선별, 견적 요청 메일 자동 등록·견적서 초안 AI 생성, 미회신 메일 탐지·리마인드, 자연어 ERP 데이터 검색
  • 데이터 이관 — 기존 엑셀(견적서·발주서·인보이스) 양식 분석 후 일괄 업로드 마이그레이션
  • 비기능 요구 — 대량 메일 동기화 시 서버 부하를 최소화하는 비동기 처리 아키텍처, 비개발자 담당자 대상 용어 설명·기획 리딩

제안 포인트

① 무역 업무 흐름을 그대로 ERP로. 선번 발행 → 견적 → 발주(PO) → 선적 → CI/PL → 재무(매입·매출/선지급·미지급) → 클레임으로 이어지는 무역 실무 동선을 데이터 모델 수준에서 설계하여, 엑셀로 흩어진 정보를 단일 흐름으로 묶습니다.

② OpenAI 실무 자동화 경험. 다국어 메일 분석·AI 문서 자동 생성·자연어 검색은 유사 규모 프로젝트에서 OpenAI/ChatGPT API를 실제 운영에 연동해 본 경험을 그대로 적용합니다. 견적 요청 메일 → 견적서 초안 자동 생성, 미회신 탐지 → 담당자 리마인드까지 업무 손실 지점을 자동화로 메웁니다.

③ 비동기 메일 동기화 아키텍처. IMAP/SMTP 연동과 대량 메일 처리를 큐 기반 비동기 워커로 분리하여, 메일 동기화 부하가 ERP 본 업무 성능에 영향을 주지 않도록 설계합니다.

④ 비개발자 눈높이 리딩. 아이디어 단계의 클라이언트를 위해 요구사항 정의서·화면 설계서를 비개발자도 읽을 수 있는 용어로 작성하고, 주간 미팅에서 의사결정 포인트를 쉬운 언어로 설명합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

부산 소재 무역/수출입 업체로서 현재 모든 업무를 엑셀로 수기 관리하고 있어, 파편화된 이메일 소통과 엑셀 관리로 인한 비효율이 누적되고 있습니다. 본 프로젝트는 선번(Project)·발주·선적·재무를 아우르는 코어 ERP를 구축하고, OpenAI API를 결합하여 다국어 메일 분석·견적서 자동 생성·자연어 검색 등 업무 자동화를 실현하는 것을 목적으로 합니다. 사내에 개발 지식 보유자가 없으므로, 기획 단계부터 비개발자 담당자의 눈높이에 맞춘 친절한 설명과 리딩을 함께 제공합니다.

  • 프론트엔드: React(Next.js) + 상용 어드민 템플릿 활용으로 개발 효율·완성도 확보
  • DB: PostgreSQL 또는 MySQL
  • 연동: OpenAI API, 사내 메일 서버 IMAP/SMTP

6. 범위 외 사항 (Out of Scope)

  • 회계 전표·세무 신고 등 전문 회계 모듈 (재무 대시보드는 포함, 정식 회계 시스템은 별도)
  • 모바일 전용 네이티브 앱 (반응형 웹으로 모바일 대응)
  • 관세청 등 대외 기관 EDI 직접 연동 (별도 협의)
  • OpenAI 외 추가 AI 모델 자체 학습·파인튜닝

일정 제안

총 120일 기준 단계별 마일스톤 및 산출물을 제안합니다. 클라이언트 희망 마일스톤(1개월 내 DB·메일 연동 / 2.5개월 내 무역 코어 ERP / 4개월 내 AI 연동·안정화)에 맞춰 설계했습니다.
